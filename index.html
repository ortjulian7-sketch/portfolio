How I work

Structured, async-ready,
no surprises.

01

Scope before pixels

I start with a structured scope breakdown — MoSCoW prioritization, database schema draft, and a realistic timeline. No surprises when we're three weeks in. No scope creep without mutual sign-off.

02

Design rooted in Bubble's logic

I design in Figma or XD with Bubble's data structure in mind. Every component maps to a real data type. Every interaction maps to a workflow. The build phase moves fast because the design phase was honest.

03

Daily structured updates

Async-first communication. I send daily structured Slack updates — what shipped, what's blocked, what's next. You always know where the project stands, without scheduling a call to find out.

04

Ship, iterate, improve

I bias toward getting something live. A working MVP in front of real users beats a perfect product that ships never. Once we're live, we iterate based on data, not assumptions.
